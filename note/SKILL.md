# Создание и улучшение заметок

## Главный тест (применять к каждой заметке)

> Заметка полезна, если будущий пользователь — уставший, в спешке — получит из неё
> ценность БЫСТРЕЕ, чем если бы учил тему заново.

Если заметка не проходит этот тест — она лишняя. Скопированное определение тест
проваливает (оно есть в гугле). Ценно то, чего в готовом виде нет: понимание,
связи, «когда применять», «где подвох».

## 6 принципов

1. **Переписывай, не транскрибируй** (тест Фейнмана). Объяснение, интуицию, «зачем» —
   ВСЕГДА своими словами: сам акт переформулирования и есть учёба.
   **Исключение — точные формулировки цитируй дословно:** утверждение теоремы,
   строгое определение, юридический/нормативный текст, сигнатуру API, формулу.
   Их искажать нельзя. Правило: **цитируй точное + объясни рядом своими словами.**
   Оформляй цитату как `> блок` или код, чтобы было видно, что это дословный источник.
2. **Одна идея — одна заметка** (атомарность). Крупные понятия выноси в отдельные
   файлы и линкуй. Признак: если заметку хочется назвать «и то, и сё» — её надо делить.
   **Лимит против переусердствования:** выноси в отдельную атомарную заметку только то,
   что **точно переиспользуется** в других темах, и не более **2-3 штук на один источник
   (лекцию/статью)**. Локальные понятия (нужны только внутри этого материала) оставляй
   **разделами** внутри основной заметки, не плоди микрофайлы. Цель — баланс «связность ↔
   не зашумить vault», а не максимум файлов.
3. **Лови WHY и WHEN, не только WHAT.** Обязательно отвечай: зачем нужно, когда
   применять, где грабли. Это главная ценность.
4. **Заголовок = утверждение или вопрос**, не ярлык. `TCP гарантирует порядок пакетов`,
   а не `TCP`. Заголовок — ручка для поиска в памяти.
5. **Связывай агрессивно.** Вставляй `[[вики-ссылки]]` на смежные понятия. Если
   ссылаешься на ещё не существующую атомарную заметку — предложи её создать.
6. **Крючок для будущего себя:** пример, грабли («тут легко запутаться, потому что…»),
   мнемоника, мини-команда.

## Процесс (как действовать)

1. **Понять источник и цель.** Лекция? CTF? Статья? Для кого заметка — для повторения
   к экзамену или для рабочего справочника? Если неясно — спроси одним вопросом.
2. **Извлечь идеи, а не предложения.** Выдели атомарные тезисы. Каждый — кандидат на
   отдельную заметку или раздел.
3. **Переформулировать своими словами.** Никакого копипаста.
4. **Структурировать по типу** (см. шаблоны ниже).
5. **Связать.** Проставить вики-ссылки; для новых понятий предложить атомарные заметки.
6. **Добавить frontmatter** (см. ниже).
7. **Отметить пробелы.** Непонятое/недоделанное помечать `🚧 TODO` или `❓вопрос`
   прямо в тексте — дырки в знании должны быть видны, а не молча теряться.

## Frontmatter (минимум)

```yaml
---
aliases: []          # альтернативные имена для поиска
tags: []             # тематические теги
created: <YYYY-MM-DD>
source:              # URL/книга/лекция — откуда
status: in-progress  # in-progress | done | review
related: []          # ["[[смежная заметка]]"]
---
```

## Шаблоны по типу заметки

### Атомарное понятие (концепция)
```markdown
# <Понятие в виде утверждения>

> Суть в 1-2 предложениях своими словами.

**Зачем:** какую проблему решает / где встречается.
**Как работает:** механизм, кратко.
**Когда применять / где подвох:** условия, грабли, антипример.
**Пример:** минимальный конкретный пример.

## Связано
- [[смежное понятие]] — чем связано
```

### Лекция (вышмат / теория) — Корнелл
```markdown
# <Курс> — Лекция N: <тема>

## Суть (заполнить ПОСЛЕ, 2-3 строки своими словами)

## Конспект
### Определение / Теорема
### Идея доказательства   ← важнее самой формулы: «индукция по n», «от противного»
### Пример

## Вопросы / TODO
- ❓...

## Связано
- [[понятие]]
```

### Процедура / CTF / how-to
```markdown
# <Задача / уязвимость>

> Что это и почему работает — 1-2 строки.

## Шаги
1. ...
   ```bash
   команда
   ```

## Ключевые факты / грабли
- ...

## Связано
- [[концепция, лежащая в основе]]
```

## Anti-patterns (не делать)

- ❌ Копировать дословно то, что нужно было ПОНЯТЬ (объяснения, интуицию). Точные
  формулировки (теорема, строгое определение, API, формула) — наоборот, цитируй верно.
- ❌ Один гигантский файл-свалка вместо атомарных заметок + связей.
- ❌ Заголовок-ярлык из одного слова.
- ❌ Заметка без единой ссылки (изолированный остров).
- ❌ Молча пропускать непонятое — помечай `❓`/`🚧 TODO`.

## После создания/улучшения

- Если в папке >5 заметок по теме и нет MOC (карты-оглавления) — предложи создать
  `_<тема> MOC.md` как точку входа.
- Кратко скажи пользователю, что было применено и какие атомарные заметки стоит
  создать следующими.
